Why Not to Use In-Line Styling

Scroll to Bottom of Page

Classmate's Page

A good web developer should not use in-line styling because of the following reasons:

  • It is hard to maintain:
    • Using in-line styling only allows the developer to edit one line at a time.
    • This causes problems when there are many pages, and the user asks for a cosmetic change, as it requires the developer to make lots of changes in lots of places rather than having all the styling in one central location.

  • In-line styling is not efficient.
    • It is time-consuming to go back and change the styling of a page one line at a time.

  • The better alternative to in-line styling would be to use a Cascading Style Sheet (CSS).
    • This allows every page to be edited at a single time in a single location.
